멀티캐스트의 데이터 전송특성
1. 특정 멀티캐스트 그룹을 대상으로 데이터를 딱 한 번 전송한다.
2. 한 번만 전송하더라도 해당 그룹에 속하는 클라이언트는 모두 데이터를 수신한다.
3. 멀티캐스트 그룹의 수는 IP주소 범위 내에서 얼마든지 추가가 가능하다.
4. 특정 멀티캐스트 그룹으로 전송되는 데이터를 수신하려면 해당 그룹에 가입하면 된다.
5. UDP를 기반으로 하기 때문에 UDP 서버/클라이언트와 구현방식이 유사하다. 
   - 목적지가 하나의 호스트가 아닌 그룹에 속한 다수의 호스트가 된다는 점에서 차이가 존재
   - 일반적인 UDP 패킷과 달리, 하나의 패킷만 네트워크 상에 띄워놓으면 라우터들은 이 패킷을 복사해서 다수의 호스트에 전달
6. 라우터의 도움으로 패킷이 그룹 내 호스트에 전달된다.
   - 패킷은 한 번만 전송하고, 라우터가 그룹에 속한 호스트들에게 패킷을 복사해서 전달한다.
7. 멀티미디어 데이터의 실시간 전송에 주로 사용된다.

# 멀티캐스트 그룹?
- 클래스 D에 속하는 IP주소(224.0.0.0 ~ 239.255.255.255)

# 터널링(Tunneling) 기술?
- 멀티캐스트를 지원하지 않는 라우터를 거쳐서 멀티캐스트 패킷을 전송하기 위한 것 

# TTL (Time to live)
- 패킷을 얼마나 멀리 전달할 것인가
- 정수로 표현되며, 라우터를 하나 거칠 때마다 1씩 감소 -> 0이 되면 더 이상 패킷이 전달되지 않는다.
- 소켓 옵션에서 'IPPROTO_IP' 레벨의 'IP_MULTICAST_TTL' 이름의 옵션을 통해 설정할 수 있다.

# 멀티캐스트 그룹으로의 가입
- 소켓 옵션에서 'IPPROTO_IP' 레벨의 'IP_ADD_MEMBERSHIP' 이름의 옵션을 통해 설정할 수 있다.

# struct ip_mreq
- @member struct in_addr imr_multiaddr: 멀티캐스트 그룹의 주소정보
- @member struct in_addr imr_interface; 그룹에 가입할 호스트의 주소정보

# Sender와 Receiver
- 멀티캐스트 기반에서는 서버, 클라이언트라는 표현 대신에 Sender, Receiver라고 표현한다.
- Sender: 멀티캐스트 데이터의 전송주체
- Receiver: 멀티캐스트 그룹의 가입과정이 필요한 데이터의 수신주체
- Sender와 Receiver 사이에서의 PORT 번호는 일치시켜야 하며, 실행 순서는 중요하지 않다.
